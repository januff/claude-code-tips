> [!tweet] @nummanali · Feb 09, 2026
> If you've been burned by Claude Code default sub agents using Haiku models
> 
> There is a very easy workaround to permanently prevent Haiku
> 
> Remap the alias via env vars in ~/.claude/settings.json:
> {
>   "env": {
>     "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-sonnet-4-5-20250929"
>   }
> }
>
> Likes: 725 · Replies: 37 · Reposts: 41

## Summary

This tip provides a workaround for preventing Claude Code's default sub-agents from using the Haiku model, offering more control over model selection. By remapping the `ANTHROPIC_DEFAULT_HAIKU_MODEL` alias through environment variables in your `~/.claude/settings.json` file, you can force the use of a different model, such as `claude-sonnet-4-5-20250929`. This allows users to avoid potentially undesirable outputs from the Haiku model and maintain consistent results with a preferred model.
